title: Enterprise Features

Better Query provides native support for complex requirements common in enterprise software.

### 🛡️ Multi-Tenancy

Isolate data automatically between customers. When enabled, every query is automatically filtered based on the user's context.

```typescript
export const invoice = createResource({
  name: "invoice",
  schema: z.object({ /* ... */ }),
  multiTenancy: {
    enabled: true,
    field: "orgId",       // Database column name
    contextKey: "orgId"   // Key in the user object (e.g. from JWT or session)
  }
});
```

The system automatically injects the tenant ID from the user object into every query.

### 📜 Policy-Based Access Control (PBAC)

Policies provide a powerful way to implement Row-Level Security (RLS). A policy can return a boolean for simple access control, or a filter object for dynamic data isolation.

```typescript
export const todo = createResource({
  name: "todo",
  policies: {
    read: async ({ user }) => {
      if (user.role === "admin") return true; // Full access
      return { ownerId: user.id };           // Automatic RLS filter
    },
    update: async ({ user, existingData }) => {
      return existingData.ownerId === user.id; // Boolean check
    }
  }
});
```

### 🗑️ Native Soft Delete

Keep your data safe with automatic soft deletion. Records are marked as deleted instead of being removed from the database.

```typescript
export const user = createResource({
  name: "user",
  softDelete: {
    enabled: true,
    field: "deletedAt" // Optional, defaults to "deletedAt"
  }
});
```

Operations like `list`, `read`, `update`, and `delete` will automatically ignore records where the soft delete field is set.

### 🎭 User Impersonation

Administrators can perform actions on behalf of other users using the `x-impersonator-id` header. This is useful for debugging or customer support.

When an impersonator ID is provided, Better Query:
- Records the impersonator in the audit logs.
- Allows the action if the impersonator has administrative privileges.
- Still applies the target user's row-level security (Ownership).

### ⚡ Custom Actions

Define business logic endpoints directly on your resources.

```typescript
export const project = createResource({
  name: "project",
  actions: {
    archive: async ({ id, adapter }) => {
      await adapter.update({
        model: "project",
        where: [{ field: "id", value: id }],
        data: { status: "archived" }
      });
      return { success: true };
    }
  }
});

// Accessible via POST /api/query/project/:id/archive
```

### ⚛️ Atomic Transactions

Execute multiple operations safely. If any operation fails, the entire transaction is rolled back automatically. The `transaction` method provides a temporary `api` object bound to the transactional context.

```typescript
import { api } from "./lib/query";

await api.transaction(async (trx) => {
  // All these operations run in a single transaction
  const project = await trx.project.create({ name: "Urgent Project" });
  await trx.task.create({ title: "Task 1", projectId: project.id });
});
```

### 📡 Real-time Synchronization

Better Query supports real-time updates via WebSockets or Server-Sent Events (SSE). The client can watch entire resources or specific records.

```typescript
import { client } from "./lib/client";

// Watch all changes on the 'project' resource
const unwatch = client.watch("project", (change) => {
  console.log("Something changed in projects:", change);
});

// Watch a specific record
client.watch("project", "project_123", (data) => {
  console.log("Project 123 was updated:", data);
});

// Stop watching
unwatch();
```

### 🔒 Field Security

Control who can see or edit specific fields.

```typescript
export const user = createResource({
  name: "user",
  schema: z.object({
    email: z.string(),
    salary: z.number(),
    role: z.string(),
  }),
  fields: {
    salary: {
      hidden: ({ user }) => user.role !== "admin", // Hidden from read operations for non-admins
      readOnly: true,                              // Read-only (cannot be mutated) for everyone
    }
  }
});
```

### 🔄 Lifecycle Hooks

Interpose custom logic at any stage of a resource's lifecycle.

```typescript
export const project = createResource({
  name: "project",
  hooks: {
    beforeCreate: async ({ data, user }) => {
      data.createdBy = user.id; // Auto-assign owner
    },
    afterDelete: async ({ id }) => {
      await notifySlack(`Project ${id} was deleted`);
    }
  }
});
```

### 🛡️ Role-Based Access Control (RBAC)

Manage permissions using scopes and ownership strategies.

```typescript
export const document = createResource({
  name: "document",
  scopes: {
    read: ["user", "admin"],
    delete: ["admin"]
  },
  ownership: {
    field: "ownerId",
    strategy: "flexible" // Owners and admins can edit
  }
});
```

### 🧹 Input Sanitization

Automatically clean your data before it reaches the database.

```typescript
export const user = createResource({
  name: "user",
  sanitization: {
    fields: {
      bio: [{ type: "trim" }, { type: "escape" }],
      email: [{ type: "lowercase" }]
    }
  }
});
```

### 🎭 GDPR Data Masking

Automatically mask sensitive data in responses.

```typescript
export const user = createResource({
  name: "user",
  masking: {
    email: (val) => val.replace(/(?<=.).(?=[^@]*?@)/g, "*"),
  }
});
```
