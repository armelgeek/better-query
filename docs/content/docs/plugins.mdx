Better Query follows a modular architecture. You can easily extend its core functionality using plugins.

### 📤 File Storage & Upload

The storage plugin allows you to handle file uploads with multiple backends.

```typescript
import { storagePlugin, LocalStorageProvider } from "better-query/plugins";

export const { handler } = betterQuery({
  // ...
  plugins: [
    storagePlugin({
      provider: new LocalStorageProvider({
        baseDir: "./uploads",
        baseUrl: "http://localhost:3000/uploads"
      }),
      uploadPath: "/upload" // Custom endpoint path
    })
  ]
});
```

#### Minio / S3 Compatible
Since Minio is S3-compatible, you can use the `S3StorageProvider`:

```typescript
import { storagePlugin, S3StorageProvider } from "better-query/plugins";

const storage = storagePlugin({
  provider: new S3StorageProvider({
    endpoint: "http://localhost:9000", // Minio Endpoint
    bucket: "uploads",
    region: "us-east-1",
    credentials: {
      accessKeyId: "minioadmin",
      secretAccessKey: "minioadmin"
    }
  })
});
```

#### Custom Provider (e.g. Cloudinary)
You can easily implement your own provider for services like Cloudinary:

```typescript
import { StorageProvider, storagePlugin } from "better-query/plugins";

class CloudinaryProvider implements StorageProvider {
  async upload({ file, path }) {
    // Use Cloudinary SDK here
    return { url: "...", key: "...", size: 0 };
  }
  async delete(key: string) { /* ... */ }
  async getSignedUrl(key: string) { /* ... */ }
}

const storage = storagePlugin({
  provider: new CloudinaryProvider()
});
```

**Usage with Fetch:**
```typescript
const formData = new FormData();
formData.append("file", myFile);

const response = await fetch("http://localhost:3000/api/query/upload", {
  method: "POST",
  body: formData
});
```

### 🔍 Audit Trail

Automatically track who changed what and when. This plugin records every create, update, and delete operation directly to the database and supports real-time queryable audit logs.

```typescript
import { auditPlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    auditPlugin({
      // Log requests and responses in audit payload
      includeRequestData: true,
      includeResponseData: true,
      // Optional: Custom console logger hook alongside DB persistence
      logger: (event) => console.log(`[AUDIT] Action: ${event.operation} on ${event.resource}`)
    })
  ]
});
```

#### Querying Audit Logs

The plugin exposes a secure HTTP endpoint at `/audit/logs` where you can query, filter, and paginate through logs:

```typescript
// Query audit logs using the Better Query Client
const logs = await client.audit.getAuditLogs({
  query: {
    resource: "project",
    operation: "update",
    page: 1,
    limit: 20
  }
});
```

### 🔒 Better Auth RLS (Row-Level Security)

Automatically secure your data based on the authenticated user. This plugin is designed to work seamlessly with [Better Auth](/docs/authentication).

```typescript
import { betterAuthRLS } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    betterAuthRLS({
      ownershipField: "userId", // The field that stores the owner ID
      adminBypass: true         // Allow admins to access everything
    })
  ]
});
```

#### Key Features
- **Automatic Filtering**: Queries are modified on-the-fly to only include records belonging to the current user.
- **Ownership Enforcement**: Prevents users from updating or deleting records that don't belong to them.
- **Auto-Assignment**: New records automatically receive the current user's ID in the ownership field.

### 🪝 Webhooks

Trigger external services when your data changes. Perfect for CI/CD triggers, Slack notifications, or syncing with third-party APIs.

```typescript
import { webhooksPlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    webhooksPlugin([
      {
        url: "https://my-service.com/webhook",
        events: ["project.create", "project.update", "delete"],
        secret: process.env.WEBHOOK_SECRET // Optional: Signs the payload
      }
    ])
  ]
});
```

### ⚡ Real-time Synchronization

Enable live data updates via WebSockets. The client can subscribe to resource-level or record-level changes automatically.

```typescript
import { realtimePlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    realtimePlugin({
      port: 3001, // Optional: Separate port for WebSockets
      path: "/realtime",
      broadcastPresence: true // Automatically share who else is watching
    })
  ]
});
```

Clients can then use `client.watch()` or `client.on()` to react to changes as they happen.

### ⚡ Caching

Improve performance by caching list and read operations. By default, it caches in memory, but supports pluggable distributed cache stores (like Redis or Keyv) for production clusters.

```typescript
import { cachePlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    cachePlugin({
      ttl: 5 * 60 * 1000, // Time to live in milliseconds (5 minutes)
      resources: ["project", "task"] // Optional: limit caching to specific resources
    })
  ]
});
```

#### Distributed Caching (Redis Example)

Simply supply a custom `store` implementation conforming to the `CacheStore` interface to swap in Redis:

```typescript
import { cachePlugin } from "better-query/plugins";
import { createClient } from "redis";

const redisClient = createClient();
await redisClient.connect();

export const { handler } = betterQuery({
  plugins: [
    cachePlugin({
      ttl: 300000,
      store: {
        get: async (key) => {
          const val = await redisClient.get(key);
          return val ? JSON.parse(val) : null;
        },
        set: async (key, val, ttl) => {
          await redisClient.set(key, JSON.stringify(val), { EX: ttl / 1000 });
        },
        delete: async (key) => {
          await redisClient.del(key);
        },
        keys: async () => {
          return await redisClient.keys("*");
        }
      }
    })
  ]
});
```

## 🕰️ Advanced History & Auditing

The History plugin provides automated record versioning and a full audit trail of changes. It allows you to see who changed what and when, and provides a simple way to restore previous versions.

### Installation

```typescript
import { betterQuery } from "better-query";
import { historyPlugin } from "better-query/plugins";

export const auth = betterQuery({
  database: { ... },
  plugins: [
    historyPlugin({
      // Optional: limit to specific resources
      resources: ["project", "organization"],
      // Optional: custom history table naming
      tableNamePattern: (model) => `${model}_versions`
    })
  ]
});
```

### Features

- **Automated Snapshots**: Automatically saves the full state of a record before updates or deletions.
- **Audit Trail**: Tracks the user ID and timestamp for every change.
- **Versioning**: Each snapshot is stored with a version number and action type (`create`, `update`, `delete`).
- **Restoration**: One-click restoration of any previous version.

### Usage in Client

The History plugin extends your resource client with specialized methods:

#### List Version History
Get a chronological list of all changes made to a specific record.

```typescript
const { data: versions } = await client.project.history("project-id");

// Each version contains:
// - snapshotData: The full state at that time
// - changedBy: The user ID who made the change
// - timestamp: When it happened
// - action: "create", "update", or "delete"
```

#### Restore a Version
Revert a record to its state at a specific point in time.

```typescript
await client.project.restore("project-id", {
  historyId: "version-uuid-from-history-list"
});
```

### Database Schema
The plugin expects a history table for each tracked resource. If you use the `autoMigrate` feature, Better Query will handle the creation of these tables automatically.

Example schema for `project_history`:
- `id`: Primary Key
- `originalId`: UUID/String (Reference to the actual project)
- `snapshotData`: JSON/Text (The serialized record)
- `action`: String
- `changedBy`: String
- `timestamp`: DateTime
- `version`: Integer

### ✦ Visual Admin Studio

Get a gorgeous, real-time database management interface directly in your application! Auto-generates columns, dynamic modals, and input forms based on your Zod schemas. Features a pulsing status indicator that listens to active WebSocket changes, live-syncing database records onto the screen instantly.

```typescript
import { adminPlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    adminPlugin({
      path: "/admin", // Access GUI at /api/query/admin
      title: "Better Query Studio"
    })
  ]
});
```

#### Key Features
- **Auto Schema Detection**: Automatically parses all Zod schemas and renders corresponding columns.
- **Dynamic CRUD Forms**: Renders inputs (toggle switch for boolean, calendar for dates, textarea for JSON blocks) based on schema attributes.
- **WebSocket Hot Reload**: Synchronizes live data changes optimistically when someone else updates a record.
- **Interactive Console**: Supports search and full create, read, update, and delete actions directly from the browser.

### ⏳ Asynchronous Background Jobs

The `jobsPlugin` provides an integrated scheduling and asynchronous execution system. It automatically registers system tables (`jobs` and `job_history`) in your database and spins up a background worker to run timed or manually-triggered tasks.

```typescript
import { jobsPlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    jobsPlugin({
      pollInterval: 60000,     // Poll for pending scheduled jobs every 60s
      defaultMaxAttempts: 3,  // Retry failed tasks up to 3 times
      enableHistory: true,    // Log execution times and results in database
      autoStart: true,        // Start polling immediately when the server boots
      handlers: {
        sendNewsletter: async (data) => {
          // Asynchronous job execution logic
          await mailer.sendNewsletter(data.campaignId);
        }
      }
    })
  ]
});
```

#### REST Endpoints Exposed
*   `POST /api/query/jobs` : Queue/schedule a new task (e.g. `schedule: "*/10 * * * *"` or `"1d"`).
*   `GET /api/query/jobs` : View planned or pending background jobs.
*   `POST /api/query/jobs/:id/trigger` : Instantly execute a specific background job.
*   `GET /api/query/jobs/:id/history` : Get task run execution history.

### 🌊 Server-Sent Events (SSE)

The `ssePlugin` provides a lightweight, one-way real-time push mechanism. It is ideal for read-only live-sync requirements where you want immediate notifications for create, update, and delete operations without setting up full bi-directional WebSockets.

```typescript
import { ssePlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    ssePlugin({
      path: "/sse" // Streams will be accessible at /api/query/sse/:resource
    })
  ]
});
```

#### Client Subscription Example
```typescript
const eventSource = new EventSource("/api/query/sse/todo");

// Listen to creation events
eventSource.addEventListener("create", (event) => {
  const newTodo = JSON.parse(event.data);
  console.log("Real-time Todo Created:", newTodo);
});
```

### 🛡️ Validation & Sanitization

The `validationPlugin` allows you to enforce strict global validation, sanitize dangerous input fields, and register custom pre-operation validation rules.

```typescript
import { validationPlugin } from "better-query/plugins";
import { z } from "zod";

export const { handler } = betterQuery({
  plugins: [
    validationPlugin({
      strict: true, // Throw ValidationError on fail. Set to false for warning-only logs.
      globalRules: {
        trimStrings: true,     // Automatically trims whitespace on all string values
        sanitizeHtml: true,    // Strips dangerous <script>, iframe, and inline JS hooks
        validateEmails: true,  // Validates any key containing the word 'email' against regex
      },
      rules: {
        user: {
          create: z.object({
            role: z.enum(["admin", "user"])
          }),
          // Custom async validation check
          customValidation: async (data, context) => {
            const errors: string[] = [];
            if (data.username === "admin") {
              errors.push("Username 'admin' is reserved");
            }
            return errors;
          }
        }
      }
    })
  ]
});
```
