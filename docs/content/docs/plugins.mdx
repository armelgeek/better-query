Better Query follows a modular architecture. You can easily extend its core functionality using plugins.

### 📤 File Storage & Upload

The storage plugin allows you to handle file uploads with multiple backends.

```typescript
import { storagePlugin, LocalStorageProvider } from "better-query/plugins";

export const { handler } = betterQuery({
  // ...
  plugins: [
    storagePlugin({
      provider: new LocalStorageProvider({
        baseDir: "./uploads",
        baseUrl: "http://localhost:3000/uploads"
      }),
      uploadPath: "/upload" // Custom endpoint path
    })
  ]
});
```

#### Minio / S3 Compatible
Since Minio is S3-compatible, you can use the `S3StorageProvider`:

```typescript
import { storagePlugin, S3StorageProvider } from "better-query/plugins";

const storage = storagePlugin({
  provider: new S3StorageProvider({
    endpoint: "http://localhost:9000", // Minio Endpoint
    bucket: "uploads",
    region: "us-east-1",
    credentials: {
      accessKeyId: "minioadmin",
      secretAccessKey: "minioadmin"
    }
  })
});
```

#### Custom Provider (e.g. Cloudinary)
You can easily implement your own provider for services like Cloudinary:

```typescript
import { StorageProvider, storagePlugin } from "better-query/plugins";

class CloudinaryProvider implements StorageProvider {
  async upload({ file, path }) {
    // Use Cloudinary SDK here
    return { url: "...", key: "...", size: 0 };
  }
  async delete(key: string) { /* ... */ }
  async getSignedUrl(key: string) { /* ... */ }
}

const storage = storagePlugin({
  provider: new CloudinaryProvider()
});
```

**Usage with Fetch:**
```typescript
const formData = new FormData();
formData.append("file", myFile);

const response = await fetch("http://localhost:3000/api/query/upload", {
  method: "POST",
  body: formData
});
```

### 🔍 Audit Trail

Automatically track who changed what and when. This plugin records every create, update, and delete operation.

```typescript
import { auditPlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    auditPlugin({
      tableName: "audit_logs", // Where to store logs
      onAudit: (log) => console.log("New change:", log)
    })
  ]
});
```

### 🪝 Webhooks

Trigger external services when your data changes.

```typescript
import { webhooksPlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    webhooksPlugin({
      endpoints: [
        {
          url: "https://my-service.com/webhook",
          events: ["project.create", "project.delete"]
        }
      ]
    })
  ]
});
```

### ⚡ Caching

Improve performance by caching list and read operations.

```typescript
import { cachePlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    cachePlugin({
      ttl: 600, // Time to live in seconds
      invalidations: [
        { resource: "project", invalidate: ["project.list"] }
      ]
    })
  ]
});
```
## 🕰️ Advanced History & Auditing

The History plugin provides automated record versioning and a full audit trail of changes. It allows you to see who changed what and when, and provides a simple way to restore previous versions.

### Installation

```typescript
import { betterQuery } from "better-query";
import { historyPlugin } from "better-query/plugins";

export const auth = betterQuery({
  database: { ... },
  plugins: [
    historyPlugin({
      // Optional: limit to specific resources
      resources: ["project", "organization"],
      // Optional: custom history table naming
      tableNamePattern: (model) => `${model}_versions`
    })
  ]
});
```

### Features

- **Automated Snapshots**: Automatically saves the full state of a record before updates or deletions.
- **Audit Trail**: Tracks the user ID and timestamp for every change.
- **Versioning**: Each snapshot is stored with a version number and action type (`create`, `update`, `delete`).
- **Restoration**: One-click restoration of any previous version.

### Usage in Client

The History plugin extends your resource client with specialized methods:

#### List Version History
Get a chronological list of all changes made to a specific record.

```typescript
const { data: versions } = await client.project.history("project-id");

// Each version contains:
// - snapshotData: The full state at that time
// - changedBy: The user ID who made the change
// - timestamp: When it happened
// - action: "create", "update", or "delete"
```

#### Restore a Version
Revert a record to its state at a specific point in time.

```typescript
await client.project.restore("project-id", {
  historyId: "version-uuid-from-history-list"
});
```

### Database Schema
The plugin expects a history table for each tracked resource. If you use the `autoMigrate` feature, Better Query will handle the creation of these tables automatically.

Example schema for `project_history`:
- `id`: Primary Key
- `originalId`: UUID/String (Reference to the actual project)
- `snapshotData`: JSON/Text (The serialized record)
- `action`: String
- `changedBy`: String
- `timestamp`: DateTime
- `version`: Integer

### 📖 OpenAPI & Swagger

Automatically generate API documentation for your resources and custom actions.

```typescript
import { openAPIPlugin } from "better-query/plugins";

export const { handler } = betterQuery({
  plugins: [
    openAPIPlugin({
      specPath: "/openapi.json",
      uiPath: "/docs", // Swagger UI will be available here
      info: {
        title: "My Enterprise API",
        version: "1.0.0"
      }
    })
  ]
});
```
