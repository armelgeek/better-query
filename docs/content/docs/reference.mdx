### betterQuery(options)

The main entry point to initialize the engine.

| Option | Type | Description |
|--------|------|-------------|
| `database` | `QueryDatabaseConfig` | Kysely instance and configuration. |
| `resources` | `QueryResourceConfig[]` | Array of resource definitions. |
| `basePath` | `string` | Base path for endpoints (default: `/api/query`). |
| `plugins` | `Plugin[]` | Array of server plugins. |
| `auth` | `AuthProvider` | Custom authentication provider. |

### QueryResourceConfig

| Property | Type | Description |
|----------|------|-------------|
| `name` | `string` | Resource name (singular, e.g., "user"). |
| `schema` | `ZodObject` | Zod schema defining the data structure. |
| `tableName` | `string` | Optional custom table name. |
| `multiTenancy` | `object` | Enable/disable data isolation. |
| `actions` | `Record` | Custom business logic endpoints. |
| `hooks` | `object` | Lifecycle hooks (before/after). |
| `scopes` | `object` | RBAC scopes for CRUD operations. |
| `ownership` | `object` | Row-level security strategy. |
| `masking` | `object` | GDPR data masking rules. |

### Universal Client Options

| Option | Type | Description |
|--------|------|-------------|
| `url` | `string` | Full URL to the Better Query endpoint. |
| `tenantId` | `string` | Global tenant ID for all requests. |
| `headers` | `object` | Custom headers to include in every request. |
| `transport` | `"ws" \| "sse"` | Real-time transport protocol. |

### Error Codes

Better Query returns standardized error codes:

- `UNAUTHORIZED`: Authentication required.
- `FORBIDDEN`: Permission denied (RBAC/Ownership).
- `NOT_FOUND`: Record not found.
- `VALIDATION_ERROR`: Zod validation failed.
- `CONFLICT`: Database constraint violation.

### Server API Invocation (`query.api`)

You can invoke CRUD endpoints and relationship queries directly in the backend code using the `query.api` object proxy. The proxy automatically supports TypeScript autocomplete typings.

#### Imbricated Calls

```typescript
// Create a new record
const project = await query.api.project.create({
  body: { name: "Nested Project" }
});

// Retrieve a list of child resources via relationship
const tasks = await query.api.project.tasks.list({
  params: { parentId: project.id }
});
```

#### Transactional Invocation

Transactions also benefit from nested object typings. Any operation made on the transaction client will be scoped to the active database transaction:

```typescript
await query.transaction(async (trx) => {
  const project = await trx.project.create({
    body: { name: "Transaction Project" }
  });

  await trx.project.tasks.create({
    params: { parentId: project.id },
    body: { title: "Subtask 1" }
  });
});
```
