This guide covers all configuration options for Better Admin, including resource setup, global settings, and theme customization.

## Basic Configuration

The `betterAdmin()` function is the main entry point for configuring your admin panel:

```typescript
import { betterAdmin, createAdminResource } from "better-admin";
import { query } from "./query";

export const admin = betterAdmin({
  query,              // Better Query instance (required)
  resources: [],      // Array of admin resources (required)
  config: {},         // Global admin configuration (optional)
});
```

## Global Configuration

The `config` object allows you to set global admin settings:

```typescript
export const admin = betterAdmin({
  query,
  resources: [/* ... */],
  config: {
    title: "My Admin Panel",
    basePath: "/admin",
    theme: {
      defaultMode: "light",
      enableModeToggle: true,
      primaryColor: "#3b82f6",
    },
    pagination: {
      defaultPageSize: 20,
      pageSizeOptions: [10, 20, 50, 100],
    },
    localization: {
      locale: "en",
      messages: {
        // Custom messages
      },
    },
  },
});
```

### Configuration Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `title` | `string` | `"Admin Panel"` | Title of the admin panel |
| `basePath` | `string` | `"/admin"` | Base URL path for admin routes |
| `theme` | `ThemeConfig` | - | Theme configuration |
| `pagination` | `PaginationConfig` | - | Default pagination settings |
| `localization` | `LocalizationConfig` | - | Localization settings |

### Theme Configuration

Customize the visual appearance of your admin panel:

```typescript
theme: {
  defaultMode: "light" | "dark" | "system",
  enableModeToggle: true,
  primaryColor: "#3b82f6",
  borderRadius: "0.5rem",
  fontFamily: "Inter, system-ui, sans-serif",
}
```

### Pagination Configuration

Set default pagination behavior:

```typescript
pagination: {
  defaultPageSize: 20,
  pageSizeOptions: [10, 20, 50, 100],
  showPageSizeSelector: true,
}
```

## Resource Configuration

Each admin resource extends a Better Query resource with UI-specific configuration:

```typescript
createAdminResource({
  name: "product",
  label: "Product",
  labelPlural: "Products",
  icon: "ðŸ“¦",
  description: "Manage products",
  list: {
    fields: ["name", "price", "status"],
    defaultSort: "name",
    defaultSortOrder: "asc",
    searchFields: ["name", "description"],
    pageSize: 20,
  },
  show: {
    fields: ["name", "description", "price", "status", "createdAt"],
  },
  create: {
    fields: ["name", "description", "price", "status"],
  },
  edit: {
    fields: ["name", "description", "price", "status"],
  },
  fieldMetadata: {
    // Field-specific configuration
  },
})
```

### Resource Options

| Option | Type | Description |
|--------|------|-------------|
| `name` | `string` | Resource name (must match Better Query resource) |
| `label` | `string` | Singular display label |
| `labelPlural` | `string` | Plural display label |
| `icon` | `string \| ReactNode` | Icon for the resource |
| `description` | `string` | Resource description |
| `list` | `ListConfig` | List view configuration |
| `show` | `ShowConfig` | Detail view configuration |
| `create` | `CreateConfig` | Create form configuration |
| `edit` | `EditConfig` | Edit form configuration |
| `fieldMetadata` | `FieldMetadata` | Field-specific metadata |

### List Configuration

Configure the list view for a resource:

```typescript
list: {
  fields: ["name", "email", "role", "createdAt"],
  defaultSort: "createdAt",
  defaultSortOrder: "desc",
  searchFields: ["name", "email"],
  filterFields: ["role", "status"],
  pageSize: 20,
  enableBulkActions: true,
  bulkActions: [
    {
      name: "delete",
      label: "Delete Selected",
      icon: "ðŸ—‘ï¸",
      confirm: true,
    },
  ],
}
```

### Show Configuration

Configure the detail view:

```typescript
show: {
  fields: ["name", "description", "price", "status", "createdAt", "updatedAt"],
  layout: "default" | "tabs" | "sections",
  sections: [
    {
      title: "Basic Information",
      fields: ["name", "description"],
    },
    {
      title: "Pricing",
      fields: ["price", "discount"],
    },
  ],
}
```

### Form Configuration

Configure create and edit forms:

```typescript
create: {
  fields: ["name", "description", "price", "status"],
  layout: "default" | "tabs" | "sections",
  defaultValues: {
    status: "draft",
    price: 0,
  },
}

edit: {
  fields: ["name", "description", "price", "status"],
  layout: "default",
  readOnlyFields: ["createdAt", "updatedAt"],
}
```

### Input Types

Better Admin supports various built-in input types:

- `text` - Single-line text input
- `textarea` - Multi-line text input
- `number` - Number input
- `email` - Email input
- `password` - Password input
- `url` - URL input
- `tel` - Phone number input
- `date` - Date picker
- `datetime` - Date and time picker
- `time` - Time picker
- `checkbox` - Single checkbox
- `select` - Dropdown select
- `multiselect` - Multi-select dropdown
- `radio` - Radio button group
- `file` - File upload
- `image` - Image upload with preview
- `richtext` - Rich text editor
- `json` - JSON editor
- `color` - Color picker

### Custom Input Types

Beyond the built-in input types, you can define custom input types to suit your specific needs. This is useful when you have specialized input requirements that aren't covered by the default types.

To use a custom input type, simply specify any string as the `inputType` and provide a `renderInput` function:

```typescript
fieldMetadata: {
  priceRange: {
    label: "Price Range",
    inputType: "price-range", // Custom input type
    renderInput: (props) => (
      <PriceRangeInput
        value={props.value}
        onChange={props.onChange}
        error={props.error}
        disabled={props.disabled}
        min={0}
        max={10000}
        currency="USD"
      />
    ),
    renderDisplay: (value) => (
      <span>
        ${value.min} - ${value.max}
      </span>
    ),
  },
  coordinates: {
    label: "Location",
    inputType: "map-picker", // Another custom type
    renderInput: (props) => (
      <MapPicker
        value={props.value}
        onChange={props.onChange}
        defaultCenter={{ lat: 0, lng: 0 }}
        zoom={10}
      />
    ),
    renderDisplay: (value) => (
      <MapDisplay lat={value.lat} lng={value.lng} />
    ),
  },
  colorScheme: {
    label: "Color Scheme",
    inputType: "advanced-color-picker", // Custom color picker
    renderInput: (props) => (
      <AdvancedColorPicker
        value={props.value}
        onChange={props.onChange}
        palette={["primary", "secondary", "accent"]}
        enableGradients={true}
      />
    ),
  },
}
```

The `renderInput` function receives props that allow you to:
- Access the current value
- Update the value via `onChange`
- Display validation errors
- Handle disabled state

You can implement any custom UI component and integrate it seamlessly with Better Admin's form system. The TypeScript types provide autocomplete for built-in types while allowing any custom string value.

### Field Options

| Option | Type | Description |
|--------|------|-------------|
| `label` | `string` | Display label for the field |
| `inputType` | `string` | Type of input control |
| `placeholder` | `string` | Placeholder text |
| `helperText` | `string` | Helper text below input |
| `required` | `boolean` | Whether field is required |
| `disabled` | `boolean` | Whether field is disabled |
| `readOnly` | `boolean` | Whether field is read-only |
| `formatter` | `function` | Function to format display value |
| `parser` | `function` | Function to parse input value |
| `validate` | `function` | Custom validation function |
| `options` | `array` | Options for select/multiselect |

## Advanced Configuration

### Custom Components

Override default components per resource:

```typescript
createAdminResource({
  name: "product",
  list: {
    component: CustomProductList,
  },
  show: {
    component: CustomProductDetail,
  },
  create: {
    component: CustomProductCreate,
  },
  edit: {
    component: CustomProductEdit,
  },
})
```

### Conditional Fields

Show/hide fields based on conditions:

```typescript
fieldMetadata: {
  discountPrice: {
    label: "Discount Price",
    inputType: "number",
    condition: (formData) => formData.hasDiscount === true,
  },
}
```

### Field Dependencies

Make fields depend on other field values:

```typescript
fieldMetadata: {
  shippingAddress: {
    label: "Shipping Address",
    inputType: "textarea",
    disabled: (formData) => formData.sameAsBilling === true,
  },
}
```
